— L'intégration Triskell Command —

Triskell Command écrit.
Pirate Life Mail envoie.
Tu regardes le log.

Quand un client achète sur ton tunnel Stripe, Triskell Command dépose une commande JSON dans la queue de Pirate Life Mail. PLM la ramasse, charge le template, insère les variables (prénom, lien de téléchargement, produit) et envoie via ton compte IMAP/SMTP habituel. Tu reçois une notification Windows. C'est tout.

Ce que ça fait concrètement.

Trois flux couverts dès l'installation. Templates éditables depuis le dossier.

01 ⚡

Livraison immédiate post-achat

Achat confirmé → email de livraison parti en moins de 30 secondes. Template delivery-v1 : prénom, nom du produit, lien de téléchargement sécurisé. Aucune intervention.

Trigger : purchase Template : delivery-v1 Délai : 0s
02 📅

Suivi J+3 automatique

3 jours après l'achat, un email de suivi part. Simple, humain, signé Jordan. Pas de CRM, pas d'outil tiers.

03 💰

Upsell J+7

Une semaine après, suggestion d'un produit complémentaire. Variables upsell_name et upsell_link configurables depuis Triskell Command.

04 🎨

Templates HTML éditables

Fichiers JSON dans %APPDATA%\PirateLifeMail\pipeline\templates\. Variables {{"{{"}}customer.name{{"}}"}}, {{"{{"}}variables.download_link{{"}}"}} et plus. Modifie, ajoute, supprime.

05 📊

— Depuis Triskell Command —

3 lignes pour déclencher une séquence complète.

from triskell_command.integrations.pirate_mail import queue_purchase_email

# Appelé depuis le webhook Stripe dans Triskell Command
queue_purchase_email(
    product        = "pack-electricien-pro",
    customer_email = "pierre.martin@gmail.com",
    customer_name  = "Pierre",
    download_link  = "https://pack-elec.triskell-studio.fr/dl/TOKEN",
    order_id       = "pi_3abc123",
)
# → Email parti en <30s. Log dans PLM mis à jour. Notification Windows.

— Questions —

Technique et pratique.

Il faut Triskell Command pour que ça marche ?

Non. Pirate Life Mail est aussi un client mail autonome — tu peux l'utiliser sans Triskell Command pour lire et écrire tes emails normalement. L'intégration pipeline est un bonus qui s'active quand les deux apps sont ouvertes.

Quel compte mail utilise-t-il pour envoyer ?

Le premier compte configuré dans PLM, ou un compte spécifique que tu désignes dans la commande (paramètre account_id). Ça passe par ton SMTP habituel — Gmail, OVH, Infomaniak, ce que tu veux.

Comment personnaliser les templates ?

Les templates sont des fichiers JSON dans %APPDATA%\PirateLifeMail\pipeline\templates\. Tu édites le HTML dans n'importe quel éditeur, tu sauvegardes, c'est pris en compte immédiatement. Variables disponibles : {{"{{"}}customer.name{{"}}"}}, {{"{{"}}product.name{{"}}"}}, {{"{{"}}variables.ta_cle{{"}}"}}.

Que se passe-t-il si PLM est fermé quand une commande arrive ?

Les commandes JSON restent dans le dossier pending/. Dès que PLM redémarre, il rattrape tout le backlog dans l'ordre. Zéro perte.

Compatible Mac / Linux ?

Windows uniquement (WinForms + WebView2). La logique IPC Python de Triskell Command est multiplateforme, mais PLM lui-même est Windows-only.
